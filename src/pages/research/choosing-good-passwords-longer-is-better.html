Skinny Passwords??? Huh???

Choosing a secure password is really hard. It's always a trade-off between using a strong password and being able to memorize your password. We know that passwords based on dictionary words and common phrases are insecure, so we rely on random password generators to create secure passwords for us.

The most secure kind of passwords are those that are completely random and use both uppercase and lowercase letters, symbols, and numbers. I like to call these fat passwords because they are made up of a random selection of all 94 printable ASCII characters, but they are never very long because few people can memorize long random passwords.

Example Fat Passwords:

$hFW3B^O)FcXtz*n}Crgc6Kq.YV85N}nn?l@

We measure password strength by the number of "guesses" it would take an attacker to figure out the password. Assuming a password is not based on dictionary words, computing it's strength is simple. The formula is:

<character set size><password length>

So if the password is randomly chosen from all 94 printable ASCII characters and is 8 characters long, it would take an attacker 948 = 6.096Ã—1015 attempts to have a 100% chance of successfully brute forcing the password.

When <character set size> is large, the password is FAT. When <password length> is large, the password is LONG. Some websites will force you to put symbols and numbers into your password but accept very short passwords. They think that short and fat passwords are more secure than long and skinny passwords. That actually isn't true. Well chosen long and skinny passwords can actually be orders of magnitude more secure than equally memorizable short and fat passwords.

Of course, when we're choosing passwords that don't have to be memorized by a human, we want them to be as long and fat as possible. When the passwords have to be memorized, it is better to use long and skinny passwords. By using long and skinny passwords, we obtain a much higher memorability to security ratio.
